# 技术分享（第 2 期）：如何让 Android 客户端具备“后端联调能力”

## 核心思路
把网络、鉴权、接口契约在客户端先标准化，避免后端一上线就大量返工。

## 三步改造
1. **Session 扩展**：不仅存登录态，还要存 AccessToken。
2. **网络基础设施**：统一 Retrofit 实例 + 鉴权拦截器。
3. **接口契约先行**：把 Auth/Feed/Topic API 固化成接口与 DTO。

## 结果
- 业务页面不需要关心 token 注入细节。
- 后端接口稳定后，客户端改动可控。
- 可持续演进到 RefreshToken、失效重试、统一错误码处理。
