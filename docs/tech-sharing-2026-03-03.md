# 技术分享（第 1 期）：把“权限规则”从界面代码里剥离出来

## 问题
当权限判断直接写在 Activity 里时，会出现：
- 页面逻辑和业务规则耦合
- 难以单测
- 后续接后端鉴权时需要重复改动

## 做法
- 用 `MainDestination` 表达业务目标。
- 用 `MainNavigationGuard` 统一判断“是否拦截”。
- Activity 只做“拿决策 -> 渲染结果”。

## 收益
1. 规则可单测：不依赖 Android 运行时。
2. 规则可复用：后续 Web 端可复用同等策略。
3. 代码更稳定：减少散落在页面里的 if-else。
